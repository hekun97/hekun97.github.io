Java提供了4种权限修饰符来修饰类以及类的内部结构，体现类及类的内部结构在被调用时可见性的大小。成员变量的4种作用域对比：

| 作用域与可见性 | 当前类 | 同包 | 父子类 | 其它包 |
| -------------- | ------ | ---- | ------ | ------ |
| public         | ✔︎      | ✔︎    | ✔︎      | ✔︎      |
| protected      | ✔︎      | ✔︎    | ✔︎      |        |
| default        | ✔︎      | ✔︎    |        |        |
| private        | ✔︎      |      |        |        |

1. public，表明该成员变量或方法对所有类或对象都是可见的，所有类或对象都可以直接访问。
2. protected，它的出现主要是为了子类能访问父类的成员变量或方法。
3. default（缺省）。表明该成员变量或方法只有自己和与其同一包内的类可见。
4. private，表明该成员变量或方法是私有的，只有当前类对其具有访问权限，除此之外的其他类或者对象都没有访问权限。

::: warning 注意

- protected与default(缺省)最难区分，protected是为了子类能访问父类的成员变量或方法，因此多了父子类的访问权限。
- 这些修饰符不能用来修饰局部变量（方法内部定义的变量）。private和protected不能用来修饰类。(只有public、abstract或final能用来修饰类)；
- 由于Java支持嵌套类，如果一个类内部还定义了嵌套类，那么，嵌套类拥有访问`private`的权限；

:::
